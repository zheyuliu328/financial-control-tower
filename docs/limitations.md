# 项目限制与边界

## 明确不做什么

### 1. 不提供真实 ERP 连接器
- 当前为模拟数据生成器
- SAP/Oracle 连接器为架构设计文档，未实际实现
- 生产使用需开发真实 API 集成

### 2. 不做合规认证
- 非 SOX/Basel/IFRS 合规系统
- 审计日志为演示性质，未经过实际审计验证
- 无监管合规认证

### 3. 不做生产级安全
- RBAC 为概念设计
- 未实施企业级加密、渗透测试
- 生产部署需专业安全审计

### 4. 不做真实欺诈检测
- 规则为示例性质
- 未在真实欺诈案例上验证
- 实际业务需根据场景调整规则

## 使用边界

| 场景 | 是否适用 | 说明 |
|:-----|:---------|:-----|
| 面试演示 | ✅ | 展示数据工程与审计思维 |
| 学习研究 | ✅ | 理解对账流程与风控规则设计 |
| 企业财务系统 | ❌ | 需专业 ERP 集成开发 |
| 监管报送 | ❌ | 需合规认证与审计 |
| 生产风控 | ❌ | 需真实数据验证与合规审查 |

## 技术限制

| 限制 | 说明 |
|:-----|:-----|
| 数据规模 | 当前使用 SQLite，大数据量需迁移至 PostgreSQL/Oracle |
| 并发处理 | 单线程实现，高并发需重新设计 |
| 实时同步 | 当前为批处理，实时同步需 CDC 方案 |

## 面试叙事建议

**可以说的**:
- "项目设计对齐财务审计思维"
- "演示了对账控制矩阵方法论"
- "面向 ERP 集成架构设计"

**不能说的**:
- "这是一个合规的财务系统"
- "已通过 SOX 审计"
- "可直接用于生产环境"
